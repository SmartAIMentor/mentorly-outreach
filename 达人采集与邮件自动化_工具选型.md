# 达人信息采集 + 自动发邮件：工具选型

> 对照《早期获客策略 v3》与当前 Mentoraixs 代码库（已有 **Resend** 做交易型邮件，**无** TikHub/FastMoss/冷邮件集成）。

---

## 一、目标流程（策略 PPT 已定）

```
采集达人 → 画像/痛点 → 生成个性化邮件或 DM → 发送+限速 → CRM 跟进
```

| 环节 | 策略文档写法 | 建议实现 |
|------|----------------|----------|
| 采集 | TikHub API：niche / 粉丝 / 互动率 | **API 优先**，落自己的 DB |
| 画像 | RecSys + Persona Graph | 自研或先用 LLM + 结构化 prompt |
| 文案 | ClawCore AI | 任意 LLM API + 模板 |
| 冷邮件 | Instantly | **Instantly** 或 **Smartlead**（勿用 Mailchimp 做冷触达） |
| 产品邮件 | （未写） | 继续用仓库内 **Resend**（注册验证、报告送达） |
| 社媒定时 | Buffer | Buffer 或 Late |

## 二、FastMoss / EchoTik / Mailchimp —— 要不要用？

### 结论速览

| 工具 | 要不要用 | 角色 |
|------|----------|------|
| **TikHub API** | ✅ 建议作为主数据源 | 程序化采集，对接自动化管道 |
| **FastMoss** | ⚠️ 可选，Phase1 人工补数 | 看榜单、找达人、导出联系思路；**一般不替代 API** |
| **EchoTik** | ⚠️ 可选，与 FastMoss 二选一即可 | 同上，避免两套付费重叠 |
| **Mailchimp** | ❌ 不推荐做冷邮件 | 适合 **已订阅** 的 Newsletter（Substack 同步 list） |
| **Instantly** | ✅ 推荐（与 PPT 一致） | 冷邮件序列、邮箱预热、多 inbox |
| **Resend**（已有） | ✅ 保留 | 验证码、诊断报告、Waitlist 确认 |

---

### 1. FastMoss / EchoTik

**是什么：** 面向 TikTok/跨境的 **达人数据分析 SaaS**（榜单、带货、粉丝画像、部分联系方式展示）。

**适合：**
- Week 1–2 **人工验证**：Bonnie/清酒快速找 50 个目标达人、看 niche 分布、抄 handle 进表格
- Agency 研究、竞品达人名单
- 国内团队熟悉中文界面时的 **侦察工具**

**不适合：**
- 作为「50 人/天全自动」的唯一数据源（通常 **无稳定开放 API**，导出/爬取有 ToS 风险）
- 与 TikHub **重复付费** 做同一件事

**建议：**
- **自动化管道：TikHub（或同类 API）→ 自己的 `creators` 表**
- **FastMoss 与 EchoTik 只选一个**，仅 Phase1 人工补邮箱、补 niche，每周导出一次即可
- 若 EchoTik/FastMoss 能导出 CSV，用脚本清洗后 **import 进 DB**，不要手抄

---

### 2. Mailchimp

**是什么：** 营销邮件 / Newsletter / 自动化旅程（**opt-in 列表**）。

**适合：**
- Substack 之外的 **邮件列表**（「Creator Economy Weekly」）
- Beta 用户 **已同意接收** 的产品更新
- 免费工具页「留邮箱」后的 **培育序列**（欢迎信 → 案例 → 邀请注册）

**不适合：**
- **冷邮件**（未许可的 agency/达人邮箱）→ 易进垃圾箱、违反 CAN-SPAM/平台政策，账号可能被封
- 替代 Instantly 的「多邮箱预热 + 冷启序列」

**建议分工：**

| 场景 | 工具 |
|------|------|
| 冷启动外联（Agency、达人商务邮箱） | **Instantly** 或 Smartlead |
| 用户注册/验证/报告 PDF | **Resend**（代码库已有） |
| Newsletter / 周更培育 | **Mailchimp** 或 **Resend Audiences** / Beehiiv |

当前仓库：`src/extensions/email/resend.ts` —— 继续用于 **交易型**，不要拿来群发冷邮件。

---

### 3. TikHub API（策略主选）

**是什么：** TikTok 等平台的 **开发者向数据 API**（用户资料、视频、互动等，按调用计费）。

**适合：**
- 按条件批量拉达人 → 写入 DB
- 免费诊断 / 报价器：用户输入链接 → 拉公开数据（与 PPT SEO 工具一致）
- 生成 DM 里的「你不知道的数据点」

**需要自建：**
- 邮箱发现：API 往往 **不直接给邮箱** → 结合 Hunter.io / Apollo / 官网 / LinkedIn / 手动补全
- 限速与合规：只发 **公开商务联系** 或 **明确 opt-in** 的地址

## 三、推荐技术栈（按阶段）

### Phase 1（Week 1–2）— 不用急着买齐 SaaS

| 动作 | 工具 |
|------|------|
| 达人名单 | 飞书多维表格 / Google Sheet |
| 人工找达人 | FastMoss **或** EchoTik（二选一）+ LinkedIn |
| 发信 | Gmail 手动 + 模板（验证转化率） |
| 记录 | 表格字段：平台、handle、粉丝、互动率、邮箱、话术版本、是否回复 |

### Phase 2（Week 3–4）— 半自动

| 动作 | 工具 |
|------|------|
| 采集 | **TikHub API** → Postgres/SQLite `creators` |
| 补邮箱 | Hunter.io / 手动 |
| 冷邮件 | **Instantly**（$30–80/月档） |
| 产品邮件 | **Resend** |
| 审核发送 | 内部 Admin 或飞书审批 → webhook 触发 Instantly |

### Phase 3（Month 2+）— 全自动

| 动作 | 工具 |
|------|------|
| 50 人/天 | Cron + 队列 + TikHub + LLM + Instantly API |
| CRM | Notion / HubSpot free / 自建 `outreach_events` 表 |
| Newsletter | Mailchimp **或** Resend Broadcast（仅 opt-in list） |

## 四、建议落库字段（飞书/DB 统一）

| 字段 | 说明 |
|------|------|
| `platform` | tiktok / instagram / youtube |
| `handle` | @username |
| `profile_url` | 主页链接 |
| `email` | 商务邮箱（可空，Phase1 手补） |
| `followers` | 粉丝数 |
| `engagement_rate` | 互动率 |
| `niche` | 垂类 |
| `country` | 地区 |
| `source` | tikhub / fastmoss / manual / agency |
| `persona_summary` | AI 画像摘要 |
| `hook_data_point` | DM 用的独家数据点 |
| `outreach_status` | pending / sent / replied / registered |
| `sequence_id` | Instantly campaign |
| `last_contacted_at` | 上次触达 |

## 五、邮件合规（必做）

1. **冷邮件**：真实发件人、物理地址（CAN-SPAM）、一键退订、停止列表
2. **不要** 用 Mailchimp/Resend 对 Purchased list 群发
3. **DM**：遵守 IG/TikTok 频率，避免同一模板刷屏
4. **诊断报告**：用户主动提交链接 = 合法处理公开数据

## 八、直接回答「需要用吗？」

| 工具 | 需要用吗？ |
|------|------------|
| **FastMoss** | 可选；Phase1 人工找达人时有用，**不必**进自动化主干 |
| **EchoTik** | 与 FastMoss 重叠，**二选一**即可 |
| **Mailchimp** | **仅** Newsletter/已订阅用户；**不要**用于冷邮件 |
| **TikHub** | **需要**（与策略一致，且支撑产品诊断） |
| **Instantly** | **需要**（冷邮件自动化，比 Mailchimp 合适） |
| **Resend** | **已有，继续用**（交易型邮件） |

**最小可行组合：** TikHub + Instantly + Resend + 飞书/表格；FastMoss/EchoTik 作人工辅助；Mailchimp 等 Substack/list 起来再加。
