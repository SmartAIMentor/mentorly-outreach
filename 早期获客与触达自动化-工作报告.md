## 二、完成的工作

### 1. 策略与运营文档

- 将 PPT 拆解为 **飞书多维表格可导入 CSV**（阶段 / 优先级 / 负责人 / KPI）。
- 输出 **达人采集 + 邮件工具选型**（TikHub / Instantly / Tally / Mailchimp / FastMoss 分工）。
- 明确 **最小可行方案**：本周不采购 Instantly/Mailchimp，用脚本 + 手动发送。

### 2. 触达自动化工具包（核心交付）

在 `scripts/outreach/` 新增 Python 工具链：

1. 读取 `creators.csv`（Instagram 主页或 @handle）。
2. 调用 **TikHub API** 拉取用户资料（粉丝、bio、邮箱）。
3. 拉取最近内容互动（优先 posts，失败则 **Reels**）。
4. 生成个性化 **DM / 冷邮件** 文案（含 Tally 链接）。
5. 导出：
   - `output/outreach_ready_*.csv` — 飞书导入
   - `output/COPY_PASTE_*.txt` — 批量复制发送
   - `output/dm/{handle}_dm.txt` — 单人 DM
   - `output/email/{handle}_email.txt` — 单人邮件（bio 含邮箱时）

### 3. 与 platform_data_fetcher 的关系

| 项目 | 说明 |
|------|------|
| 仓库 | `gh repo clone SmartAIMentor/platform_data_fetcher` |
| 已有能力 | `instagram_user_fetch.py` + `tikhub==2.1.1`，IG 资料/帖子/可选 Gemini 视频分析 |
| 缺口 | 无批量触达、无 DM/邮件模板、无 Tally 集成；TikTok 批量搜索未实现 |
| 本仓库做法 | 独立脚本 `batch_outreach.py`，复用同款 SDK，并适配 TikHub **新响应结构**（`data` 直接为用户对象；Reels 为 `data.items`） |

⚠️ **安全：** `platform_data_fetcher` 内曾硬编码 API Key，勿再使用；统一用本目录 `.env` 中的 `TIKHUB_API_KEY`（勿提交 git）。

### 4. 联调与修复记录

| 问题 | 处理 |
|------|------|
| 旧 TikHub Key 401 | 更换用户提供的新 Key |
| `get_user_posts` HTTP 400 | 自动降级为 `get_user_reels` |
| `data.user` 为空、粉丝显示 0 | 增加 `parse_profile_user()` 兼容扁平 `data` |
| Reels 返回 `items` 非 `edges` | 更新 `reel_edges_to_items()` |
| 爆款帖互动率 >100% | `build_hook()` 对高互动改用品牌向表述 |
| 姓名含 `—` 符号 | `display_first_name()` 提取 Ramona 等有效名 |
